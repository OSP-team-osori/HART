# [Project Overview] Agent-Harness: TADD 기반 AI 에이전트 신뢰도 검증 파이프라인

## 1. 프로젝트 탄생 배경 (Why we build this)
바야흐로 AI 코딩 어시스턴트(Claude, Aider, GitHub Copilot 등)의 시대입니다. 하지만 AI가 작성한 코드는 완벽하지 않습니다. 문법적으로는 그럴싸해 보이지만 실제로는 논리적 오류나 보안 취약점을 포함하는 '그럴싸한 환각(Plausible Hallucination)' 현상이 빈번하게 발생합니다.

현재 개발자들은 AI가 코드를 짜주면, 터미널을 열고 직접 빌드 및 테스트 명령어를 입력하여 결과물이 제대로 동작하는지 눈으로 확인해야 합니다. 이 반복적인 수동 검증 과정은 CI/CD(지속적 통합/배포) 파이프라인의 심각한 병목이 되고 있습니다.

우리는 이 문제를 해결하기 위해, 교수님께서 강의에서 강조하신 TADD(Test + AI Driven Development) 방법론을 완벽하게 자동화하는 시스템 인프라를 구축하고자 합니다.

## 2. Agent-Harness란 무엇인가? (What is it?)
Agent-Harness는 AI 코딩 에이전트의 작업 결과물을 인간의 개입 없이 기계적으로, 그리고 즉각적으로 검증하는 '자동화된 관제 및 평가 플랫폼'입니다.

개발자가 '코드 구현자'에서 '시스템 아키텍트 및 품질 관리자'로 역할이 진화하는 현대 소프트웨어 공학의 트렌드에 맞춰, 개발자는 오직 요구사항(Spec)과 테스트 코드(RED)만 정의합니다. 이후 AI가 코드를 작성(GREEN)하면, Agent-Harness 시스템이 백그라운드에서 이를 실시간으로 감시하고, 작업 완료 즉시 하네스 테스트(단위 테스트)를 돌려 그 결과를 평가합니다.

## 3. 핵심 가치 (Core Values)
- **AI 환각의 원천 차단 (Zero-Trust AI):**
  AI가 생성한 코드를 맹신하지 않습니다. 시스템이 강제로 실행하는 단위 테스트(Unit Test)를 통과해야만 '유효한 코드'로 인정받습니다.
- **테스트 주도 개발의 자동화 (Automated TADD):**
  개발자가 직접 터미널에서 pytest를 칠 필요가 없습니다. AI 프로세스의 생명주기를 서버가 가로채어 검증 파이프라인을 자동으로 트리거합니다.
- **데이터 기반 신뢰도 평가 (Reliability Scoring):**
  테스트 통과율, 소요 시간, API 토큰 소비량 등의 데이터를 수집하여, 해당 AI 에이전트가 얼마나 '믿을 만한 주니어 개발자'인지 객관적인 수치로 증명합니다.

## 4. 시스템 동작 시나리오 (User Journey)
본 프로젝트는 복잡성을 덜어내고 본질에 집중한 MVP(Minimum Viable Product) 형태로 구동됩니다.

1. **[명령 하달]:** 개발자가 대시보드 웹 UI에서 요구사항(예: "비밀번호 검증 로직 구현해줘")을 입력하고 실행 버튼을 누릅니다.
2. **[실시간 관제]:** 서버(Spring Boot)가 격리된 Docker 컨테이너 내부에서 Aider(Gemini API 연동, 여력 되면 claude code등) 에이전트를 구동합니다. AI가 고민하고 코드를 작성하는 모든 터미널 출력(Log) 과정이 대시보드에 실시간(SSE)으로 중계됩니다.
3. **[자동 검증]:** AI가 코드 작성을 마치고 종료되면, 서버가 즉각적으로 해당 코드에 대한 단위 테스트 스크립트를 실행합니다.
4. **[결과 보고]:** 테스트의 Pass/Fail 여부가 대시보드에 시각화되며, 사용된 토큰 비용과 함께 데이터베이스에 영구 기록됩니다.
   *(※ 대시보드에는 단일 메인 에이전트의 구동 화면 외에도, 향후 시스템이 '멀티 에이전트 관제 플랫폼'으로 확장될 수 있음을 보여주는 세련된 더미(Dummy) 슬롯들이 제공됩니다.)*

## 5. 왜 이 프로젝트가 특별한가? (Project Highlight)
단순한 '웹 게시판'이나 '단순 API 호출 앱'을 넘어, 본 프로젝트는 운영체제(OS) 레벨의 프로세스 제어와 실시간 스트림(I/O) 처리를 다루는 백엔드 시스템 엔지니어링의 정수를 보여줍니다.

- `java.lang.ProcessBuilder`를 활용한 자식 프로세스 생명주기 관리
- OS 출력 버퍼링 제어 및 정규식을 활용한 실시간 로그 파싱
- WebSocket보다 가벼운 Server-Sent Events(SSE) 기반의 단방향 실시간 데이터 푸시
- 환경 오염이 없는 격리된 Docker 컨테이너 기반의 Runner 구조 설계
- **Docs as Code 기반 협업:** 기획자(PM)가 요구사항을 단순 줄글이 아닌 pytest 기반의 테스트 코드와 더미 데이터(JSON)로 직접 깃허브에 커밋(Commit)하여, 개발자들과 명확한 코드로 소통하는 실무형 협업 프로세스 구현

Agent-Harness는 개발팀이 귀찮은 검증 작업에서 해방되어, 오직 '아키텍처 설계'와 '비즈니스 로직 구상'에만 집중할 수 있도록 돕는 가장 든든한 DevOps 파트너가 될 것입니다.
